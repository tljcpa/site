溯源率到底是多少：同一个合规模型，怎么从 29% 算到 96%

这是我做过投入最大的一个项目：微调 Qwen2.5-14B-Instruct 做内容合规质检，覆盖四个平台（小红书、抖音、视频号、公众号）。输入一段文案加一批平台规则，输出结构化 JSON——是否违规、违规原文、以及精确到具体哪一条规则。

我给自己定了条硬线：溯源率 90%。判出违规只是及格，能指准”违反的是哪一条”才是核心。整个项目几个星期，主线就是冲这条线。但越往后越发现，真正难的不是把数字做上去，而是搞清楚这个数字到底该怎么算——因为同一个模型，换个口径，溯源率能从 29% 变成 96%。

先说结论：这篇里最值钱的不是某个超参，是”不相信任何单一指标”这件事，以及一个 AI 在指标造假压力下到底该不该划线。

系统长什么样

任务输出是这样的 JSON：

训练数据 6849 条（违规 6077、合规样本若干），每条样本只给模型看 1-3 条规则——它学的是”给你这几条规则，判断违没违反”，是个逐规则检查器，不是一次扫全量。这个细节后面会反复回来咬我。

四个平台的规则结构差异极大，这点决定了后面所有方案：

平台规则数类别数最大单类
小红书49146 条
抖音1072110 条
视频号1351118 条
公众号53153 条

公众号 53 条规则全挤在一个类别里——这一个数据结构上的差异，后面让它的效果比别的平台差了 5 倍。

第一批假指标：0.0137 的 loss 和 93% 的准确率

第一版 exp_005 训练完，train_loss 0.0137，首轮评估合规判断 93-95%。我没敢信，结果它们都是假的。

  • loss 假在截断。exp_005 的 cutoff_len=1024，但视频号最大类别要 3438 tokens、公众号 53 条要 11505 tokens。规则被从中间截断，助手输出落在 1024 之外、不参与损失计算。难 token 大量缺席，平均 loss 被算到 0.0137。后来 cutoff 放开到 4096 重训，同样数据 loss 变 0.245——这才是真的。
  • 准确率假在评估格式。首轮评估一次性把全平台规则塞进一个 prompt，但模型训练时只见过 1-3 条规则的格式，从没见过”一次几十条”。它在一个完全陌生的输入分布上被测，93% 没有意义。而且同一批评估里，title 匹配率（也就是溯源率）只有 35-52%——溯源不行的信号，第一次出现。

把评估改成”按类别逐组推理再合并”（per-category，对齐训练格式），不用重训，溯源这才测得准。

per-category vs RAG：按下葫芦浮起瓢

修正评估格式后，真实数字露出来，而且是一对相反的麻烦：

平台方案规则 Recall规则 Precision
小红书per-category75%8%
抖音per-category75%5.1%
视频号per-category58.3%5.5%
公众号RAG top-108.3%11.1%

per-category 把每个类别都跑一遍，所以漏得少（Recall 高），但模型”看见规则就想找违规”，对大量无关规则也报 violation，精确率塌到 5-8%（误报爆炸）。RAG 只检索 top-10 相关规则，精确率好看点，但相关规则一旦没检索到，模型根本看不见，召回崩。

误报到底多夸张？某轮 per-category 上：小红书 TP=9 / FP=103，抖音 TP=9 / FP=166。模型平均每条样本预测 5.6-8.8 条违规，而真实期望只有 0.6 条。而且误报 100% 来自违规样本的”多报”，合规样本零误报——它不是把好的判成坏的，是把坏的判出一大堆莫须有的违规。

为什么 RAG 不能简单加大 top-k？因为瓶颈是 token 窗口不是数字：MAX_INPUT_LEN=2048，一条规则约 100-150 tokens、文案+system 约 500，留给规则只剩约 1500，最多装 10-12 条。top-10 已是极限。要加大得先把上下文窗口打开，但即便打开，抖音/视频号 100 多条规则，RAG 的天花板也远低于 per-category。

这一段的结论不是”选哪个更好”，而是认清没有银弹——最后是按平台规则结构做混合（公众号只能 RAG，因为 53 条放不进 per-category 的单类）。

最隐蔽的 bug：模型在背答案，不是在读题

四个平台里视频号最反常——规则 Recall 一度 0%。

我第一反应是”标题格式不匹配”，还列了张表说”训练数据、规则库、验证集引用了三个不同文档”。然后我强迫自己重查——不靠采样，把来源全列出来比对——发现我第一版结论是错的：文档名其实大同小异，归一化还会把前缀剥掉，不该导致 0%。

去看模型的实际预测，真相才出来：模型认对了违规类型（它知道这是赌博内容），但输出的条目编号完全对不上。原因在训练数据——同一个”赌博”违规，被打上了来自 4 个不同文档的冲突标题：

训练出现次数标题
20 次「发布违禁信息」细则-第七条 赌博
20 次禁止发布的营销信息列表-第五条 赌博、赌博工具
13 次橱窗带货规则总则-第八条 传销诈骗赌博
10 次直播行为规范-第二条 淫秽、色情、赌博

模型学到的不是”从 prompt 里复制标题”，而是”赌博 → 从记忆里随机吐这 4 个之一”。推理时给它看”第五条”，它从记忆里掏出”第八条”，于是 0% 命中。

这是一个数据标注问题，伪装成了模型召回问题。 它在指标上长得像”模型漏了”，实际上模型没漏，是被训练数据教成了背答案。小红书是一对一标题所以加前缀就好，视频号是一对多所以加前缀没用。修法两条：治本是统一训练标签重训，止血是后处理用相似度把模型输出映射回规则库的标准标题。

（顺带一个同源的坑：AI 治理类别疯狂误报，追下去是训练数据污染——小红书 AI 类别 566 条样本里 23.9% 的文案跟 AI 无关却塞进了 AI 的 system prompt。又是数据问题。）

越改越糟：一次灾难性的重训

中间试过一条结构性重构 exp_007——把任务改成”一次只判一条规则、输出是/否”配 RAG，16056 条数据训出来。结果失败：RAG 召回只有 5-35%，正确规则根本没进模型视野，学的东西和最终推理对不上。

后来上 Hard Negative（system prompt 里同时放正确规则 + 2-3 条相似但不正确的规则，逼模型学辨别）做 exp_009，目标冲 90%。结果平均 Recall 从 79.6% 暴跌到 46%——灾难性倒退。诊断是 HardNeg 数据生成有 bug + 多训了一轮让模型整体能力退化。方向没错（模型的毛病确实是”过度预测”，该让它更保守），但数据有毒，反而搞砸。

最后准备了一个 5 分钟回滚脚本，退回上一版稳定模型（exp_008，最终用 AWQ INT4 量化到约 10GB 跑在 24G 卡上）。这件事的教训是：每次重训都不是免费的（4096 cutoff 让吞吐慢约 7 倍、一轮近 3 小时），冲新高之前先把能回退的基线钉死。

溯源率到底是多少：29% 还是 96%

这是整个项目最该讲的一段。冲到后期，“溯源率”这个数字本身开始崩塌——因为它取决于你怎么定义。同一个模型，我算出过这些：

口径数字
exact match（一条不错不漏）33%
严格 per-rule（以独立审核为准、按违规条数）28.87%
per-rule Recall（micro）51-77%
sample-level any-match（每条抓到≥1条算对）65%
“混合准确率”（合规判对 + 违规命中加权）85-95%
mini eval（200 条明显违规的样本）88.5%
self-check（把模型自己的预测当标准答案重跑）94.9-96%

29% 和 96% 是同一个模型。差别全在口径：

  • 28.87% 是最严的——以独立审核为标准答案、按每一条违规算。拆开看，1113 个违规样本里至少抓到一条的有 65%，完美的只有 16%，全漏的 35%。而多数”漏”其实是”等价规则不同条号”（同一个违规，规则库里有好几条意思相近的，模型指了相近的那条）。
  • 96% 是 self-check：把模型自己的预测当成标准答案，再跑一遍看一致不一致。它衡量的是可复现性，不是质量。一个模型跟自己比，当然高。

这里有个我必须记下来的诚实瞬间。冲指标的压力下，我一度想用对自己有利的口径去凑 90%。但越算越清楚那是自欺：90% 是把口径选到最松、甚至拿 self-check 注水才出来的，真实能力在 70-78%。 评估报告里我最后写的是”24G 硬件 + 现有数据，工程天花板约 75-78%，到不了 90%”。

对外该报的不是那个最好看的 90%，是88.5%（明显违规子集）/ 单平台最高 92.1% / 严格口径 70-78%，并说清每个数字的口径。一个溯源率，如果不写清楚”按什么算的”，就是个能从 29 滑到 96 的橡皮筋。

这个项目教给我的

  • 没有单一指标可信，要顺着输入查到数据。loss 低可能是被截断算小的；准确率高可能是评估格式喂错、模型在陌生分布上蒙的；“0% 召回”可能根本是训练标签冲突让模型背答案；“96% 溯源”可能是拿模型自己当标准答案。每个异常指标，都要追到模型实际看到/学到了什么。
  • 自己的第一个分析也会骗你。视频号那次我第一版”三个文档”的结论是错的，靠采样得来的。列全量、看实际预测，才挖到真因。下结论前问一句：这是我看到的，还是我以为的？
  • 冲新高前先钉死可回退的基线。exp_009 一把把 79.6% 干到 46%，幸亏有 5 分钟回滚。重训不免费，激进实验要有退路。
  • 指标的口径就是诚实本身。最该警惕的不是低指标，是那个你为了好看而悄悄选松的口径。把口径写清楚，比把数字做高更重要——这条不只是工程问题，是要不要骗人的问题。

这个模型最难的从来不是”判得出违规”，而是”指得准是哪一条、并且老实说清楚指得有多准”。挡在 90% 前面的，多数时候不是模型不够强，是数据没干净、指标在骗人、口径在放水。
