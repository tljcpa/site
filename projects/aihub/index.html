AI Hub：多渠道 AI 平台

个人

一个开箱即用的多渠道 AI 平台：对话、文生图、用户系统、卡密充值、管理后台全都有。后端把多个模型渠道统一成一套接口，支持多 Key 轮询和负载均衡。

一个完整的多渠道 AI 平台，自己一个人前后端做下来的。一句话概括：把市面上各家模型（OpenAI、Claude、智谱、硅基流动等任意 OpenAI 兼容接口）统一接进来，对外提供对话、文生图、用户系统、卡密充值和一整套管理后台。有在线演示站，注册送 10 次体验额度。

核心设计：把”渠道”抽象出来

这个平台工程上最关键的抽象是渠道（channel）。不同厂商的模型 API 各不相同，但大多兼容 OpenAI 格式，所以我把每个上游统一抽象成一个渠道，对内暴露一套接口。在这之上做了两件实用的事：

  • 多 Key 轮询：一个渠道可以挂多个 API Key，请求在这些 Key 之间轮转。单 Key 有速率限制或额度用尽时，多 Key 轮询能摊开压力、提高可用性。
  • 负载均衡：渠道之间自动均衡，避免所有流量压在一个上游。

对用户来说，他只是在选”模型”；底层走哪个渠道、用哪个 Key，平台自己调度。这是这类聚合平台真正的价值所在——不是接一个模型，而是把一堆不稳定的上游，聚合成一个稳定的对外服务。

用户侧功能

  • 专栏广场：按领域预设好 system prompt，一键进入专属对话（代码助手、写作助手、翻译等），降低普通用户的使用门槛。
  • 多渠道对话：流式输出（打字机效果）、多轮对话、Markdown 渲染、代码高亮、Artifacts 预览。
  • 对话管理：新建/删除/重命名会话，消息编辑/重试/复制，对话导出分享。
  • 图片生成：文生图、图生图，支持负面提示词和 Steps/CFG/Seed 这些高级参数，带历史记录和收藏。
  • 用户系统：注册/登录（可开邀请码准入），卡密激活充值，按”次数 + 有效期”双维度管理额度。

管理侧（双重认证保护的后台）

后台是这个项目工作量很大的一块，因为一个能运营的平台，光有用户功能不够：

  • 渠道管理：增删改渠道，配多 Key，自动负载均衡。
  • 模型管理：全局启用/禁用指定模型、改显示名、按渠道过滤。
  • 卡密管理：批量生成卡密（次数 + 有效期两个维度），查使用状态。
  • 邀请码管理：独立于卡密，单独控制注册准入。
  • 用户管理：调余额/有效期/角色，一键封禁解封。
  • 使用统计、公告系统：按天看调用量，发全站公告。
  • 全局配置在线编辑：直接在后台改 config.json，无需重启就能切功能开关——这点对线上运营很重要，改个配置不用停服。

小结

这是我做过的最完整的一个全栈产品：从用户能用的对话/绘图，到运营需要的渠道调度、卡密、统计、热配置，前后端一条龙。做下来最大的体会是，“接个模型出对话框”只是开头，真正撑起一个能用的平台的，是渠道抽象、多 Key 调度、后台运营这些不显眼但缺一不可的工程。
